🔐 auth-guard.js 導入指南

✅ 這份文件說明如何把現有每個頁面的 onAuthStateChanged 改成一行 authGuard()，以及 permissions.html 的部署方式。

Step 1 — 部署兩個新檔案到 repo

把以下兩個檔案放到 GitHub repo 根目錄（與其他 .html 同層）：

檔案說明部署後 URL
auth-guard.js 集中式權限守衛，所有頁面共用 /hr-system/auth-guard.js
permissions.html 後台圖形化權限管理介面 /hr-system/permissions.html

Step 2 — 初始化 Firestore 權限設定

第一次打開 permissions.html（用 admin 帳號登入），直接按「儲存變更」，系統就會把預設權限寫入 Firestore settings/permissions。之後就用這個介面管理，不需要改程式碼。

⚠️ 記得把 permissions.html 加進 dashboard.html 的 ALL_MODULES 陣列，讓它出現在首頁選單。

Step 3 — 改造各頁面（每頁約改 5 分鐘）

每個頁面只需要做兩件事：① 刪掉自己的 Firebase 初始化 + onAuthStateChanged；② 加一行 import + 一個 authGuard() 呼叫。

以 meeting.html 為例

❌ 改前（各頁自己判斷）
import { initializeApp } from "...firebase-app.js";
import { getAuth, onAuthStateChanged,
         signOut } from "...firebase-auth.js";
import { getFirestore, doc,
         getDoc } from "...firebase-firestore.js";

const firebaseConfig = { apiKey: "...", ... };
const app  = initializeApp(firebaseConfig);
const auth = getAuth(app);
const db   = getFirestore(app);

onAuthStateChanged(auth, async user => {
  if (!user) {
    window.location.href = 'index.html';
    return;
  }
  const snap = await getDoc(doc(db,'users',user.uid));
  if (!snap.exists() || snap.data().disabled) {
    await signOut(auth);
    window.location.href = 'index.html?disabled=1';
    return;
  }
  const userData = snap.data();
  const role = userData.role || 'employee';

  // 各頁自己的 role 判斷邏輯（每頁都不一樣）
  if (role === 'employee') {
    window.location.href = 'dashboard.html';
    return;
  }
  const locs = userData.locations || [];
  currentRole = role;
  userLocations = locs;
  // ... 繼續初始化
});
✅ 改後（一行搞定）
import { authGuard, db } from './auth-guard.js';

// 就這樣，不需要其他 Firebase import
// 不需要 firebaseConfig
// 不需要 onAuthStateChanged
// role/locations 判斷全部集中在 auth-guard.js

authGuard('meeting', ({
  user, role, locations, userData, db
}) => {
  // 直接從這裡開始寫功能邏輯
  currentRole   = role;
  userLocations = locations;

  loadRecords();
});

// 新增功能時：
// 1. 在 permissions.html 新增一列
// 2. 這裡的 'meeting' 換成新 key
// 完成

未來新增功能的流程

1

建立新的 xxx.html，在 script 最上面加 import { authGuard, db } from './auth-guard.js';，呼叫 authGuard('xxx', callback)。

2

打開 permissions.html，在 ALL_MODULES 陣列新增一筆（icon、name、key、desc），設定哪些 role 可見，儲存。

3

在 dashboard.html 的 ALL_MODULES 陣列也加同一筆，讓它出現在首頁卡片和側邊欄。

4

完成。不需要在功能頁內寫任何 role 判斷邏輯。

💡 注意：改造完成後，dashboard.html 的 VISIBLE_KEYS 物件也可以改成從 Firestore 讀取，這樣首頁卡片的顯示也會跟著 permissions.html 的設定走，完全統一。需要的話可以再做這一步。
